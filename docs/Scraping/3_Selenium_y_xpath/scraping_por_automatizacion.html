13.1. XPath (XML Path Language)¬∂

Un selector muy utilizado es el Xpath, basado en un lenguaje dise√±ado para localizar elementos en un archivo XML o HTML.

Es una manera de recorrer un √°rbol como por ejemplo el DOM (document object model).

Por ejempllo si vamos a la p√°gina del Instituto Humai y presionamos f12 veremos esto:

Existen dos maneras de acceder a un elemento usando Xpath:

13.1.1. XPath absoluto¬∂

  • Contiene la ruta completa desde el elemento raiz hasta el elemento de interes.
    Ejemplo: /html/body/section[2]/div[2]/a[2]/div/div[1]/img

    • Contra: Cualquier cambio en la ruta del elemento hace que ya no se acceda de esa forma.

13.1.2. XPath relativo¬∂

  • Comienza haciendo referencia al elemento que queremos ubicar en relaci√≥n con una ubicaci√≥n particular. Esto significa que el elemento est√° posicionado con relaci√≥n a su posici√≥n normal.
    Ejemplo: //*[@id="que-hacemos-a-tag"]/div/div[1]/img

    • Cualquier cambio en el dise√±o de la p√°gina o la jerarqu√≠a DOM tendr√° un impacto m√≠nimo (o nulo) en el selector XPath existente.

    • Su estructura b√°sica es la siguiente:

    //nombreDelTag[@Atributo="valor"]

13.1.3. Seleccion de nodos desconocidos¬∂

  • El asterisco (*) o wildcard sirve como reemplazo para el nombre del tag, funcionando como comod√≠n para cualquiera de ellos.
    Ejemplo: //*[@id="soy_un_div"]

  • El arroba seguido de asterisco (@*) matchea cualquier atributo de un elemento.
    Ejemplo: //h2[@*="soy_un_subtitulo"]

13.1.4. Algunas funciones¬∂

  • contains() permite ver si un elemento contiene una string en particular. No es necesario que sea match exacto, sino que forme parte de la string.
    Ejemplo: //div[contains(@class,"que")]

  • text() nos permite obtener un elemento en base al texto que tiene dentro del tag
    Ejemplo: //*[text()="SABER M√ÅS"]

Tambien los podriamos usar juntos: //*[contains(text(),"SABER")] ü§Ø

Hay mucho mucho m√°s!

def procesar_mails(pagesource_list):
